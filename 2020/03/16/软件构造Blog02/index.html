环境与配置

在开始使用新软件的时候会遇到一些令人疑惑的问题，在这里记录一些我自己遇到的小问题。

Java Build Path

第一个小问题是开始的时候错误地为项目添加了JUnit5的jar包，但是后来想要去除更改的时候遇到了很大的问题，最后通过更改Java build path来解决这个问题。
这里说明一下Java build path，它就是用于设置在编译时能够用到的Java项目和依赖的类，分为几个小项：源码文件夹、项目、类库。这几个很好理解就是编译时用到的一些源。但是还有两个项目，一个是Order and Export，还有一个是Module Dependencies。

  • Order and Export：
    order指的是各种项目的依赖顺序，而Export指的是在本项目被别的项目依赖的时候，只有exported的源才能被别人利用到
  • Module Dependencies：
    是在高版本java中，可以用来可视化操作各个项目间依赖关系，目前没有用到，将来用到的时候可以研究研究。

eclipse GUI

eclipse其实还是蛮难看的，于是进行了一些theme的安装，安装了MoonRise之，选择了Solarized Dark之后遇到了一个问题，就是package explorer的颜色变得非常地不合群了，于是被迫进行了以下的调整。

  • Preference - General - Appearance - Colors and Fonts
    在里面对一些参数进行修改

但是左边的一小条仍然非常难受，其实也是可以设置的

  • Range indicator color
    考验色彩的时候到了，怎么选都和 Solarized 或者sublime 的主题不搭，直到最后选择了黑色才算是合群了一些。

github

因为在上学期就使用过GitHub，所以在开始的时候没有遇到特别大的困难。最大的困难可能还是在于网速不够快，clone到本地的速度也是问题之一。
但是在接下来使用的过程中就会遇到一些乱七八糟的问题

  • git config –usr.email “” 的时候输错了导致自己上传的commit没有出现在contribution里
    解决方案：
  • 能否不通过GitHub的接口创建repository，然后直接在git bash中通过git remote add添加仓库并进行上传

这就是前期遇到的一大堆琐碎的问题，很多在遇到的时候非常地错愕与混乱，也花费我一定的时间在网上寻找答案，希望接下来的课程可以不必为这些小问题所困扰。
